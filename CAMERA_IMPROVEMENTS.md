# Улучшения работы с камерой

## Обзор

В этот релиз включены значительные улучшения работы с камерой в интерфейсе NextUI, которые решают проблемы с изменением разрешения на лету и обработкой ошибок подключения.

## Решенные проблемы

### 1. Разрешение не менялось на лету
**Проблема**: При изменении разрешения камеры поток видео переставал работать.

**Решение**: 
- Добавлен механизм принудительной перезагрузки изображения через timestamp
- Используется параметр `?t=${timestamp}` в URL для принудительной перезагрузки
- Состояние ошибки сбрасывается автоматически при смене разрешения
- Оставлены только два оптимальных разрешения: VGA (640×480) и SVGA (800×600)

### 2. Поток видео отваливался при смене URL
**Проблема**: При изменении IP адреса камеры поток видео переставал работать, и ошибка оставалась даже после возврата к рабочему URL.

**Решение**:
- Добавлен новый обработчик `handleBaseUrlChange` для смены базового URL
- При смене URL сбрасываются все состояния ошибки и подключения
- Добавлен ключ для принудительного перерендеринга изображения

### 3. Ошибка подключения оставалась после возврата к рабочему URL
**Проблема**: После возникновения ошибки подключения, даже при возврате к рабочему URL, ошибка оставалась.

**Решение**:
- При любой смене URL (базового или разрешения) сбрасывается состояние ошибки
- Добавлен механизм автоматического переподключения при ошибках

## Новые функции

### Автоматическое переподключение

Система теперь автоматически пытается переподключиться к камере при возникновении ошибки:

- **Максимум 3 попытки** переподключения
- **Пауза 2 секунды** между попытками
- **Визуальный индикатор** процесса переподключения
- Автоматический сброс счетчика попыток при успешном подключении

### Индикаторы статуса подключения

Добавлены визуальные индикаторы, которые показывают текущее состояние подключения:

- ✅ **Подключено** - камера успешно подключена
- ❌ **Ошибка подключения** - не удалось подключиться к камере
- 🔄 **Переподключение...** - система пытается переподключиться
- ⏳ **Подключение...** - идет процесс первоначального подключения
- Счетчик попыток подключения (например, "Попытка: 1/3")

### Улучшенная обработка ошибок

- Кнопка "Попробовать снова" появляется только после 3 неудачных попыток
- При достижении лимита попыток пользователю предлагается вручную попробовать снова
- Все состояния ошибки автоматически сбрасываются при смене URL или разрешения

## Технические изменения

### Новые состояния (state)

```typescript
const [isReconnecting, setIsReconnecting] = useState(false);
const [connectionAttempts, setConnectionAttempts] = useState(0);
const [timestamp, setTimestamp] = useState(Date.now());
```

### Новые refs

```typescript
const imageRef = useRef<HTMLImageElement>(null);
const reconnectTimeoutRef = useRef<NodeJS.Timeout | null>(null);
```

### Принудительная перезагрузка через timestamp

Вместо использования ключа для принудительного перерендеринга, мы добавляем timestamp к URL:

```typescript
const streamURLWithTimestamp = useMemo(() => {
  return `${customStreamURL}?t=${timestamp}`;
}, [customStreamURL, timestamp]);
```

Этот подход гарантирует, что браузер будет загружать новое изображение при каждом изменении timestamp.

### Новые функции

#### `handleResolutionChange(newResolution: string)`
Обрабатывает изменение разрешения камеры:
- Сбрасывает состояние ошибки
- Сбрасывает счетчик попыток
- Обновляет разрешение и URL потока
- Обновляет timestamp для принудительной перезагрузки

#### `handleBaseUrlChange(newBaseUrl: string)`
Обрабатывает изменение базового URL камеры:
- Сбрасывает все состояния ошибки
- Сбрасывает счетчик попыток
- Обновляет базовый URL
- Обновляет timestamp для принудительной перезагрузки

#### `handleUpdateURL()`
Принудительно обновляет URL потока:
- Сбрасывает все состояния ошибки
- Сбрасывает счетчик попыток
- Обновляет timestamp для принудительной перезагрузки

### Автоматическое переподключение

```typescript
useEffect(() => {
  if (imageError && connectionAttempts < 3) {
    setIsReconnecting(true);
    reconnectTimeoutRef.current = setTimeout(() => {
      setConnectionAttempts(prev => prev + 1);
      setImageError(false);
      setTimestamp(Date.now());
      setIsReconnecting(false);
    }, 2000); // Пауза 2 секунды перед переподключением
  }

  return () => {
    if (reconnectTimeoutRef.current) {
      clearTimeout(reconnectTimeoutRef.current);
    }
  };
}, [imageError, connectionAttempts]);
```

## Использование

### Смена разрешения

1. Выберите желаемое разрешение из выпадающего списка
2. Система автоматически:
   - Сбросит состояние ошибки
   - Обновит URL потока
   - Попытается подключиться к новому URL

### Смена IP адреса камеры

1. Введите новый IP адрес в поле "IP адрес камеры"
2. Нажмите кнопку "Обновить"
3. Система автоматически:
   - Сбросит все состояния ошибки
   - Обновит базовый URL
   - Попытается подключиться к новому адресу

### Обработка ошибок

При возникновении ошибки:

1. **Автоматическое переподключение**: Система автоматически попытается переподключиться (до 3 попыток)
2. **Визуальные индикаторы**: Вы увидите статус подключения и счетчик попыток
3. **Ручное переподключение**: После 3 неудачных попыток появится кнопка "Попробовать снова"

## Рекомендации по использованию

### Выбор разрешения

- **VGA (640×480)**: Стандартное разрешение, рекомендуется для большинства случаев
- **SVGA (800×600)**: Высокое качество, требуется более быстрое соединение

### Устранение неполадок

**Камера не подключается:**
1. Проверьте правильность IP адреса
2. Убедитесь, что камера включена и подключена к сети
3. Попробуйте уменьшить разрешение
4. Проверьте, что порт (по умолчанию 81) доступен

**Видео заикается или медленно загружается:**
1. Попробуйте уменьшить разрешение
2. Проверьте качество Wi-Fi соединения
3. Убедитесь, что камера и устройство в одной сети

**Ошибка остается после смены URL:**
1. Нажмите кнопку "Обновить" после ввода нового URL
2. Если ошибка сохраняется, попробуйте перезагрузить страницу
