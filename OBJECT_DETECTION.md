# Детекция объектов в камере

## Обзор

Добавлена функциональность компьютерного зрения для автоматического определения объектов в видео потоке с камеры. Используется модель COCO-SSD (Common Objects in Context) на базе TensorFlow.js с двумя конфигурациями:

1. **COCO-SSD Lite** - быстрая конфигурация для слабых устройств
2. **COCO-SSD Accurate** - точная конфигурация для максимальной точности

## Технические требования

- Современный браузер с поддержкой WebGL или WebAssembly
- Минимум 2 ГБ оперативной памяти
- Стабильное интернет-соединение для загрузки модели (при первом запуске)
- Поддержка Canvas API

## Сравнение конфигураций

| Характеристика | COCO-SSD Lite | COCO-SSD Accurate |
|---------------|-----------------|---------------------|
| Архитектура | lite_mobilenet_v2 | mobilenet_v1 |
| Точность | Хорошая | Высокая |
| Скорость | Очень быстрая | Быстрая |
| Размер модели | ~2 МБ | ~3 МБ |
| Классы | 80 | 80 |
| Backend | TensorFlow.js | TensorFlow.js |
| Рекомендуется для | Быстрая детекция, слабые устройства | Максимальная точность, мощные устройства |

## Тестирование детекции

Для проверки работоспособности детекции объектов была добавлена специальная вкладка **"🧪 Тест ИИ"** на главной странице.

### Как использовать тест:

1. Запустите проект: `npm run dev`
2. Откройте главную страницу
3. Перейдите на вкладку **"🧪 Тест ИИ"**
4. Загрузите любое изображение с объектами (люди, животные, транспорт, бытовые предметы)
5. Нажмите кнопку **"🔍 Детектировать"**
6. Просмотрите результаты детекции:
   - Прямоугольники вокруг обнаруженных объектов
   - Список обнаруженных объектов с классом и уверенностью

### Логирование в консоли:

Компонент ObjectDetector выводит подробную информацию в консоль браузера:

- `🔄 Начинаем инициализацию TensorFlow.js...` - начало загрузки
- `✅ TensorFlow.js backend готов: webgl/wasm/cpu` - backend готов
- `🔄 Загружаем модель COCO-SSD...` - загрузка модели
- `✅ Модель COCO-SSD загружена успешно` - модель готова
- `🔍 Начинаем детекцию объектов...` - начало детекции
- `✅ Детекция завершена, обнаружено объектов: N` - количество объектов
- `📦 Объект N: класс (уверенность%)` - информация о каждом объекте
- `📤 Передаем N объектов в родительский компонент` - передача результатов

### Диагностика проблем:

Если детекция не работает:

1. **Откройте консоль разработчика** (F12 или Cmd+Option+I)
2. **Перейдите на вкладку "🧪 Тест ИИ"**
3. **Загрузите тестовое изображение**
4. **Нажмите "🔍 Детектировать"**
5. **Проверьте логи в консоли**:
   - Если видите `✅ Модель COCO-SSD загружена успешно` - модель работает
   - Если видите `✅ Детекция завершена, обнаружено объектов: 0` - модель работает, но объектов не найдено
   - Если видите ошибки - проверьте детали ошибки в логах

6. **Проверьте изображение**:
   - Убедитесь, что изображение содержит распознаваемые объекты (люди, животные, транспорт и т.д.)
   - Проверьте качество изображения (хорошее освещение, четкость)
   - Попробуйте загрузить другое изображение

## Использование детекции с видео потоком

### Выбор конфигурации модели

1. Перейдите на вкладку **"📷 Камера"**
2. Убедитесь, что камера подключена и видео поток работает
3. Нажмите кнопку **"🤖 Детекция объектов"** → **"❌ Выключено"**
4. Кнопка изменится на **"✅ Включено"**
5. Появится выпадающий список для выбора конфигурации:
   - **COCO-SSD Lite (быстрая)** - максимальная скорость, подходит для слабых устройств
   - **COCO-SSD Accurate (точная)** - высокая точность, рекомендуется для мощных устройств
6. Выберите нужную конфигурацию

### Рекомендации по выбору конфигурации

- **Используйте COCO-SSD Lite**, если:
  - У вас слабое устройство (менее 4 ГБ RAM)
  - Нужна максимальная скорость детекции
  - Достаточно хорошей точности

- **Используйте COCO-SSD Accurate**, если:
  - У вас мощное устройство (4+ ГБ RAM)
  - Нужна максимальная точность детекции
  - Важна детекция маленьких объектов
  - Работаете в сложных условиях освещения

### Просмотр результатов

После включения детекции:

1. **Bounding Boxes**: Прямоугольники будут отображаться вокруг обнаруженных объектов на видео потоке

   - Класс объекта (например, "person", "car")
   - Уверенность детекции в процентах

### Выключение детекции

Нажмите кнопку "✅ Включено" для выключения детекции. Кнопка изменится на "❌ Выключено".

## Технические детали

### Архитектура

**COCO-SSD Lite (TensorFlow.js)**:
- **Библиотека**: TensorFlow.js
- **Модель**: COCO-SSD с архитектурой lite_mobilenet_v2
- **Интервал детекции**: 500мс
- **Рендеринг**: Canvas API для отрисовки bounding boxes

**COCO-SSD Accurate (TensorFlow.js)**:
- **Библиотека**: TensorFlow.js
- **Модель**: COCO-SSD с архитектурой mobilenet_v1
- **Интервал детекции**: 500мс
- **Рендеринг**: Canvas API для отрисовки bounding boxes

### Производительность

- Первая загрузка модели занимает несколько секунд
- Детекция выполняется асинхронно без блокировки UI
- Canvas отрисовывается поверх изображения без влияния на производительность
- COCO-SSD Accurate обеспечивает более высокую точность, но работает немного медленнее

### Цветовая кодировка

Каждый класс объекта имеет свой уникальный цвет для удобства визуализации. Примеры:
- 🔴 Человек (person): #FF6B6B
- 🔵 Автомобиль (car): #4ECDC4
- 🟡 Собака (dog): #F39C12
- 🟢 Кошка (cat): #E74C3C

## Troubleshooting

### Детекция не работает с видео потоком камеры

Если детекция работает на тестовых изображениях, но не работает с видео потоком камеры:

1. **Проверьте статус подключения к камере**:
   - Убедитесь, что камера подключена и видео поток активен
   - Проверьте статус: "✅ Подключено"

2. **Проверьте консоль браузера**:
   - Откройте консоль разработчика (F12)
   - Ищите сообщения с префиксом `⏸️ Детекция не запущена:`
   - Проверьте значения: `hasModel`, `hasImage`, `enabled`, `hasCanvas`

3. **Возможные причины**:
   - `hasImage: false` - изображение камеры не загружено
   - `enabled: false` - детекция выключена (нажмите кнопку включения)
   - `hasModel: false` - модель еще не загрузилась (подождите несколько секунд)

4. **Решение**:
   - Убедитесь, что видео поток работает (видите изображение с камеры)
   - Включите детекцию кнопкой "🤖 Детекция объектов"
   - Подождите загрузки модели (появится сообщение "✅ Модель COCO-SSD загружена успешно")

### Детекция не находит объекты

Если детекция работает, но не находит объекты:

1. **Проверьте качество изображения**:
   - Хорошее освещение
   - Четкое изображение
   - Объекты должны быть видны на видео

2. **Попробуйте тестовое изображение**:
   - Перейдите на вкладку "🧪 Тест ИИ"
   - Загрузите изображение с известными объектами
   - Если тест работает - проблема в видео потоке
   - Если тест не работает - проблема в модели или браузере

3. **Проверьте логи в консоли**:
   - Ищите `🔍 Начинаем детекцию объектов...`
   - Ищите `✅ Детекция завершена, обнаружено объектов: N`
   - Если `N = 0` - модель не нашла объектов (нормально для пустой сцены)

### Модель не загружается

- Проверьте интернет-соединение
- Очистите кэш браузера
- Попробуйте перезагрузить страницу
- Убедитесь, что браузер поддерживает WebGL или WebAssembly
- Проверьте консоль браузера на наличие ошибок

### Ошибка "No backend found in registry"

Эта ошибка была исправлена в текущей версии. Если она все еще появляется:
- Обновите зависимости: `npm install`
- Перезапустите dev сервер: `npm run dev`
- Очистите кэш браузера
- Убедитесь, что используете современный браузер (Chrome, Firefox, Safari, Edge)

### Детекция не работает

- Убедитесь, что камера подключена и видео поток активен
- Проверьте, что детекция включена (кнопка "✅ Включено")
- Улучшите освещение сцены

### Низкая производительность

- Закройте другие вкладки браузера
- Уменьшите разрешение видео потока
- Используйте более мощное устройство
